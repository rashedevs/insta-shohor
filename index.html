Bonus section

As we know, javascript is a synchronous, single-threaded, interpreted language. It can run through any modern browser engine or Node.js runtime. The Execution context of the engine is consist of two main components, a heap & a call stack. Thus, it can do one thing at a time. The javascript engine parse the code line by line then convert it to the machine code to execute.

Using event loop, javascript can work in asynchronous way. If a task takes too long to complete, other tasks can be halted & could slow down our application. To avoid such implications, javascript has a concurrency model based on the event loop that has the ability to process multiple tasks asynchronously. Event loop keeps a relation with call stack & callback queue. Checks if the call stack is free, then lets know the callback queue to passes the callback function to be executed.

Local storage & Session storage are two objects of web storage which allows to save key/value pairs in the browser. Both can store user information in a similar way. The main difference is the expiration date. Local storage keeps store the user data without expiration date, this data will not be deleted when user closed the browser windows. It will be available for day, week, month and year. For session storage, it will delete all (data) windows when browser tab/windows closed by a web user.
